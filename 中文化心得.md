# EOB1 中文化心得 — 從 EXE hex patch 到 ScummVM source patch

## 起點

最初想法是模仿 EOB2 中文版（隱月傳奇）的做法 — 直接 patch DOS `EOB.EXE` + `MCGA.OVL` 加 Big5 渲染 hook。試了好幾天，遇到一堆雷：

- 16-bit MZ overlay 內 jump 距離限制
- EOB 執行期會把 hook 區域當 scratch buffer 覆寫
- Big5 endianness（subset BE-stored vs x86 LE-read）
- 不同 OVL (MCGA/CGA/EGA/TGA) 各有 hook
- 字模沒地方塞，subset 要 binary search

雖然最後做到了，但只能 8x8 byte-pair 像素中文，看不清楚；要 16x14 真中文又有 hook 位置被覆寫的問題。整個技術棧脆弱、難維護。

## 轉折

切到 ScummVM 路線後，**3 天的工作量縮成一個下午**。原因：

1. **ScummVM 已經有 EOB2 中文版的完整 infrastructure**
   - `Graphics::Big5Font` class — 整套 Big5 字模渲染
   - `ChineseTwoByteFontEoB` — 自動 wrap 任意 ASCII font 加 Big5 lead/trail 偵測
   - `Big5Font::loadPrefixedRaw` — `ceob.pat` 載入器
   - 90 個 EOB2 中文 string provider 為 template

2. **只需要把 GI_EOB2 only 的 gating 放寬到 GI_EOB1**

3. **字串走 ScummVM resource system**，不用碰 EXE byte offset

## 字模選擇

### V1: 直接用 EOB2 16x14 CHINFONT.FNT crop

EOB2 字模有 12,811 字 (16x14 XOR 加密 + 28 byte/glyph)。Python 解密 + 重新編碼成 ScummVM `ceob.pat` 格式 (BE codepoint + 28 byte bitmap)。
- ✅ 跑得起來
- ❌ 14 row 太高，CAMP menu 各項目 overlap

### V2: BoutiqueBitmap9x9 (9px)

OFL 1.1 開源 12,858 字 TTF。
- ✅ 不缺字
- ❌ 9px 用 freetype-py render 出來模糊不清

### V3 (現在): EOB2 16x12 crop + BoutiqueBitmap fallback

```python
HEIGHT = 12        # 從 14 砍上下各 1 row
TOP_TRIM = 1
# rows [1..13)
```

12,811 字從 EOB2 來（清晰銳利）+ 940 字從 BoutiqueBitmap 12px 補（如 `憶` `謄` 之類 EOB2 沒有的字）= 13,751 字 / 357KB。

`tools/build_ceob_combined.py` 一次產出。

## 翻譯來源

### 大宗：先前 EOB1 EXE patch 專案的 `full_patches.json`

402 條 EOB.EXE byte offset → Big5 字串對照。`tools/gen_eob1_chinese.py` 讀 ScummVM 的 `eob1_dos_english.h`，逐字串到 lookup 表查中文，產出 `eob1_dos_chinese.h`。

自動匹配率：**380/446 = 85.2%**。

短字標籤（STR/INT/HP/yes/no/ok/NPC names）full_patches.json 沒有，手寫 38 條：

```json
    "CON": "體",

```

最終：**416/446 = 93.3%**（剩 30 條是 `%d/%d` 之類格式碼，不該翻）。

## 踩過的雷

### 1. create_kyradat 的 need-list 是 per-(game, platform, special)

不分語言。如果我加 `kEoB1IntroStringsTower` 到 `eob1FloppyNeed[]`，EN_ANY 就要有 provider，但英文版根本沒這個 string provider（intro 是 baked CPS 圖）。這讓 intro narration 翻譯沒辦法走 ScummVM strings 系統。

解法（未實作）：在 ScummVM source 加 `kSpecial=5` Chinese-fan 變種，分離 need-list。

### 2. `kEoBBaseManDef` (need id 318) 不分語言但需要

ManDef 是 manual protection 的 byte 查表（120 bytes），langSpecific=true 但每個語言版本實際資料一樣。Fix：register English provider 為 ZH_TWN 別名：
```cpp
{ kEoBBaseManDef, kEoB1, kPlatformDOS, kNoSpecial, ZH_TWN, &kEoB1ManDefDOSEnglishProvider },
```

### 3. 字模高度太高 → CAMP 菜單疊行

14-row 字 + EOB1 chargen Y-spacing 12px → 行間 overlap。改 12-row 後 OK。

### 4. WSLg `[WARN:COPY MODE]` 視窗看不到

WSLg 在某些 GPU 上 fallback software rendering，視窗存在於 `msrdc.exe` hwnd 但畫面不 paint 到 Windows desktop。Workaround：`wsl --shutdown` 重啟 WSLg；長期：cross-compile to Windows native。

### 5. PowerShell 在 .bat 內的 here-string

用 `^` 接行的 PowerShell here-string 在 cmd.exe 內會被分行解析錯誤，殘餘 tokens 被當 cmd 命令執行。解法：用單行 `powershell -Command "{...}"` 或乾脆寫成獨立 `.ps1`。

### 6. Native scummvm.exe 跨平台 build

MinGW-w64 cross-compile from Ubuntu 22.04 + SDL2-mingw devel binary，跑：

```bash
./configure --host=x86_64-w64-mingw32 --backend=sdl \
    --with-sdl-prefix=/opt/sdl2-mingw/SDL2-2.30.7/x86_64-w64-mingw32 \
    --disable-all-engines --enable-engine=kyra,eob \
    --disable-translation --disable-flac --disable-vorbis \
    --disable-mad --disable-fluidsynth --disable-freetype2 \
    --disable-jpeg --disable-png --disable-mt32emu
make -j8
```

產出 99MB binary，strip 後 27MB。+ SDL2.dll (2.3MB) + KYRA.DAT (2MB) + ceob.pat (357KB) = ~32MB 完整 portable 包。

## 沒做的東西（與為何沒做）

| 項目 | 卡在哪 |
|---|---|
| 中文名字輸入 (IME) | EOB2 `CHINFONT.COD` 是注音 IME 字典但結構非標 (混 4-byte 注音碼 + 1-byte count + 後段 Big5 區), 且 ScummVM 沒有 IME widget 框架 |
| Intro narration 中文 | need-list 設計（見踩雷 #1）|
| 物品 / 怪物名中文 | 在 `EOBDATA*.PAK` 內 (Westwood PAK 格式)，要 in-place patch 或加 ZH_TWN provider 走 need-list 重構 |
| 對話 / 關卡文字 | 在 `LEVEL*.INF` 內，類似 |
| 過場圖內文字 | baked CPS 圖 (`TEXT.CMP`)，要 image edit |

## 比較表：EXE patch 路線 vs ScummVM 路線

| | EXE patch (放棄) | ScummVM patch (現在) |
|---|---|---|
| 工作量 | ~3 天 | 半天 + iteration |
| 跨平台 | DOS only | Windows / Linux / Mac / iOS / Android (ScummVM 全平台) |
| 維護 | 不可能 | 可 PR 上游 |
| 字模 | 8x8 byte-pair 或 16x14 hook | 任意 height，wrap 在 ASCII font 上 |
| 翻譯 | byte offset，slot size 限制 | 任意長度 string |
| 偵測 | 不適用 | ScummVM 自動偵測 |

教訓：**找對工具是 80% 的工作**。逆向 1991 年 DOS overlay 之前，先看看有沒有 modern engine 已經吃掉這個遊戲。

## 工具棧

- **Reverse**: Python 3 + cp950 codec
- **Source patch**: 5 個 C++ 檔，diff <1000 行
- **Build (Linux)**: `gcc`, `g++`, GNU make
- **Build (Windows cross)**: `mingw-w64-x86-64-posix` + SDL2 mingw devel
- **Runtime (Windows)**: SDL2.dll + scummvm.exe
- **CI (沒用)**: ScummVM 上游 GitHub Actions Windows matrix（fork 沒自動觸發）
